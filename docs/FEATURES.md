## 1. 서비스 개요

**정담**은 결혼식·장례식 등 경조사를 디지털로 관리하는 플랫폼이다.  
주최자(호스트)는 앱에서 경조사를 만들고, 하객은 QR 코드 혹은 링크를 통해 웹에서 방명록을 남기고 축의금/조의금을 전달한다.

| 구분 | 기술 스택 | 역할 |
|------|-----------|------|
| **앱** (GyeongjoApp) | React Native + Expo (SDK 53) | 호스트용 - 경조사 생성·관리 |
| **웹** (contribution-web) | Next.js 14 | 하객용 - 청첩장 열람·방명록·축의금 |
| **백엔드** | Supabase (PostgreSQL + Realtime + Storage) | 공용 |
| **SMS** | Twilio | 인증 코드 발송 |

## 2. 인증 시스템

### 2-1. 앱 (호스트)
- **휴대폰 SMS 인증** (Primary)
  - Twilio로 6자리 인증 코드 발송
  - 인증 코드 5분 만료, 시도 횟수 제한
  - AsyncStorage + Supabase 세션 하이브리드 저장
- **OAuth 로그인** (Backup - 현재 비활성화)
  - Google / Kakao via Supabase Auth
- **세션 유지**
  - 앱 재시작 시 AsyncStorage에서 세션 복원
  - Supabase 세션 만료 시 자동 갱신

### 2-2. 웹 (하객)
- **휴대폰 SMS 인증**
  - 방명록 작성·축의금 제출 시 본인 확인용
  - 인증 완료 후 `verifiedPhone` localStorage 저장
  - 동일 브라우저에서 재방문 시 인증 생략
- **이벤트별 세션 분리**
  - `arrival_checked_{eventId}` 키로 청첩장마다 독립 세션 관리
  - A 청첩장에서 인증해도 B 청첩장에서는 별도 확인

### 2-3. 테스트 계정
- `010-5835-9358` → SMS 없이 인증번호 `999999` 자동 처리

### 3-1. 결혼식 생성
| 입력 항목 | 내용 |
|-----------|------|

| 장소 | 웨딩홀명, 상세 주소, 주차 안내 |
| 연락처 | 주 연락처, 보조 연락처 |
| 사진 | 메인 사진, 갤러리 사진 (신랑·신부 구분) |
| 커스텀 메시지 | 자유 입력 |
| 프리셋 금액 | 하객이 선택할 축의금 금액 버튼 설정 |
| 관계 정의 | 신랑측/신부측 관계 유형 커스텀 |

### 3-2. 장례식 생성
| 입력 항목 | 내용 |
|-----------|------|
| 고인 정보 | 성함, 나이, 성별 |
| 일정 | 사망일, 입관일, 발인일 |
| 장소 | 장례식장, 장지·봉안당 위치 |
| 상주 정보 | 상주 성함 및 관계 |

- Premium 플랜: 무제한

## 4. 청첩장 · 부고장 템플릿

하객이 링크/QR로 접속했을 때 보이는 디지털 청첩장 화면.  
모바일 전용 디자인, 호스트가 생성 시 선택.

| 템플릿 ID | 이름 | 특징 |
|-----------|------|------|
| `classic-elegant` | 클래식 엘레강스 | 크림·골드 톤, 전통적 우아함 |
| `romantic-pink` | 로맨틱 핑크 | 핑크·로즈 계열, 화사한 분위기 |
| `modern-dark` | 모던 다크 | 다크 배경, 세련된 고급감 |
| `modern-minimal` | 모던 미니멀 | 화이트 베이스, 깔끔한 미니멀 |
| `elegant-garden` | 오로라 블랙 | 어두운 배경, 골드 포인트 |
| `warm-orange` | 웜 오렌지 | 따뜻한 오렌지·베이지 톤 |
| `korean-elegant` | 클린 화이트 | 밝고 깔끔한 화이트 계열 |
| `vintage` | 빈티지 | 복고풍 디자인 |
| `korean` | 한국 전통 | 전통 문양, 한지 느낌 |
| `garden` | 엘레강트 가든 | 자연·식물 모티프 |

### 템플릿 공통 기능
- **인트로 애니메이션**: 청첩장 봉투 열리는 오프닝 효과
- **이미지 슬라이드쇼**: 업로드된 사진 자동 전환
- **실시간 방명록 목록**: 다른 하객이 남긴 방명록 표시
- **실시간 토스트**: 새 방명록/축의금 알림 팝업
- **축의금 참여 바텀시트**: WelcomeChoiceModal (하단에서 올라오는 선택 UI)
- **OG 이미지 자동 생성**: 카카오톡 공유 시 커플 이름·날짜·장소 표시
- **장소 지도 연동**: 주소 클릭 시 지도 앱 연결

### 장례식 템플릿 (1종)
- 절제되고 경건한 디자인, 다크 계열

## 5. QR 코드 시스템

### 흐름
```
호스트 (앱) → QR 코드 생성
    ↓ 인쇄 또는 공유
하객 → QR 스캔
    ↓
웹 청첩장 오픈 → 방명록·축의금 입력
    ↓ 실시간
호스트 앱 → 즉시 반영
```

### 기능
- 이벤트별 고유 QR 코드 자동 생성
- QR 링크: `https://contribution-web-srgt.vercel.app/template/{eventId}?template={templateId}`
- QR 이미지 저장 (스크린샷)
- 링크 복사 / 공유 (Share API)

## 6. 방명록 시스템

### 하객 (웹)
- **입력 플로우**
  1. WelcomeChoiceModal에서 "방명록 + 축의금" 선택
  2. 휴대폰 번호 입력 → SMS 인증
  3. 방명록 내용 작성 (이름, 메시지)
  4. 자동으로 축의금 입력으로 이어짐

- **입력 항목**: 이름, 메시지, 공개/비공개 여부
- **중복 방지**: 동일 번호로 재접속 시 기존 내용 수정 가능
- **실시간 반영**: 작성 즉시 청첩장 화면에 표시

### 호스트 (앱)
- 전체 방명록 목록 조회
- 개별 항목 수정·삭제
- 공개/비공개 토글

### API
| 엔드포인트 | 기능 |
|------------|------|
| `GET /api/get-guestbook` | 이벤트별 방명록 조회 |
| `POST /api/submit-guestbook` | 방명록 제출 |
| `PUT /api/update-guestbook` | 방명록 수정 |
| `DELETE /api/delete-guestbook` | 방명록 삭제 |
| `GET /api/check-guestbook-duplicate` | 중복 확인 |
| `GET /api/get-my-guestbook` | 내 방명록 조회 |

## 7. 축의금 · 조의금 관리

### 하객 (웹) - 입력 플로우
단계별 바텀시트 모달로 진행 (프로그레스 바 표시)

| 단계 | 내용 |
|------|------|
| 1단계 | **방명록 책자** - 3D 애니메이션 책자, 랜덤 이름 163개 표시 (재미 요소) |
| 2단계 | **이름 입력** |
| 3단계 | **금액 입력** - 프리셋 버튼 + 직접 입력 |
| 4단계 | **측 선택** - 신랑측 / 신부측 (결혼식) |
| 5단계 | **관계 선택** - 가족, 친구, 직장동료 등 |

- 입력 완료 후 수정 가능 (편집 모드)
- "내가 축의한 금액" 섹션 하단 고정 표시

### 호스트 (앱)
- 전체 축의금 목록 조회
- 금액 합계 자동 집계
- 관계별·측별 분류
- 개별 항목 수정·삭제

### 관계 유형 (39개 정의)
- 결혼식: 신랑측 / 신부측 각각 → 가족, 친척, 친구, 직장동료, 선배, 후배, 이웃, 기타
- 장례식: 고인과의 관계 별도 정의

## 15. 보안

- **RLS (Row-Level Security)**: 사용자별 데이터 격리, 이벤트 소유자만 수정 가능
- **SMS 인증**: 인증 코드 5분 만료, 시도 횟수 제한
- **서비스 롤 키 분리**: 클라이언트는 anon key, 서버 API는 service role key
- **인증 코드 자동 정리**: `cleanup_expired_sms_verifications()` DB 함수

## 16. 데이터베이스 구조

### 핵심 테이블

| 테이블 | 설명 | 주요 필드 |
|--------|------|-----------|
| `users` | 사용자 정보 | id, phone, name, subscription_type, max_events |
| `events` | 경조사 이벤트 | id, user_id, event_type, event_date, groom_name, bride_name, location, template_style, status |
| `guest_book` | 방명록 + 축의금 통합 | id, event_id, guest_name, guest_phone, relation_category, relation_detail, amount, message, is_public, is_verified |
| `personal_schedules` | 개인 일정 | id, user_id, title, event_type, event_date, reminder_time |
| `relation_types` | 관계 유형 코드 (39개) | id, event_type, category, detail, display_name |
| `sms_verifications` | SMS 인증 기록 | id, phone, verification_code, expires_at, is_verified, attempts_count |
| `event_messages` | 공개 메시지 | id, event_id, message, is_public |

### 뷰 및 함수
- `events_with_title`: 이벤트 제목 호환 뷰
- `get_monthly_events()`: 월별 통계
- `get_user_stats()`: 사용자 대시보드 통계
- `get_event_contribution_stats()`: 이벤트별 수금 통계
- `update_updated_at_column()`: 수정 시간 자동 업데이트
- `cleanup_expired_sms_verifications()`: 만료 인증 코드 정리
